# Release Process - ITHCSoftwareApp

## Versioning Strategy
We follow Semantic Versioning:
MAJOR.MINOR.PATCH

## Development Workflow
- Developers push code to GitHub
- Jenkins pipeline runs automatically

## Build Process
- Install dependencies
- Run tests
- Build frontend
- Package application

## Release Process
- Assign version number
- Generate release artifacts
- Create GitHub release

## Deployment
- Deploy application to Dev environment

## Rollback
- Revert to previous version if issues occur
